\begin{abstract}

Modern virtualization environments often face significant performance overheads when utilizing Virtual Input/Output Memory Management Units (vIOMMUs). This paper evaluates vFree, an optimized suite of page unmapping strategies designed to mitigate these bottlenecks which has been shown to achieve optimal throughput while maintaining security. Specifically, we characterize the performance of this method when running in multi-tenant environments which allows a realistic evaluation of how vFree performs.

Our results demonstrate that for transmit (TX) flows, the vFree strategy achieves optimal performance, matching the aggregate throughput of systems with the vIOMMU disabled across configurations ranging from 1 to 16 VMs.  However, we identify a significant performance discrepancy in the receive (RX) flows, where the aggregate throughput decreases as a function of the number of VMs. This work provides a comparative analysis of paging strategies in cases with multiple VMs and highlights the remaining challenges. 

\end{abstract}
